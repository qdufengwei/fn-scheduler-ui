# 任务监控页面需求规格说明书

## 一、页面概述

### 1.1 页面定位
任务监控页面是监控系统模块下的子页面，展示正在运行的任务的CPU和内存资源使用情况，帮助用户和运维人员监控任务运行状态，排查资源使用异常。

### 1.2 页面功能
- 展示当前运行任务的基本信息（名称、状态、类型、运行时长）
- 展示任务的CPU使用量
- 展示任务的内存使用量
- 支持按任务类型和任务名称筛选
- 支持时间范围查询
- 支持自动刷新
- 支持报表下载

## 三、任务信息展示区

### 3.1 当前任务信息

| 信息项 | 说明 |
|--------|------|
| 任务名称 | 当前查看的任务名称 |
| 状态 | 任务运行状态：运行中、排队中、停止等 |
| 类型 | 任务类型：CPU、Simple、MPI等 |
| 已运行时长 | 任务已运行的时间，格式：XXh XXm XXs |

### 3.2 报表下载

| 按钮 | 功能 |
|------|------|
| 报表下载 | 导出当前任务的监控数据报表 |

---

## 四、控制区功能

### 4.1 切换任务

| 筛选项 | 控件类型 | 说明 |
|--------|----------|------|
| 任务类型 | 下拉选择框 | 按任务类型筛选：CPU、Simple、MPI等 |
| 任务名称 | 下拉选择框 | 选择具体的任务名称 |

### 4.2 筛选条件

| 筛选项 | 控件类型 | 说明 |
|--------|----------|------|
| 数据聚合方式 | 下拉选择框 | 选择数据聚合方式：平均值、最大值、最小值等 |

### 4.3 自动刷新设置

| 设置项 | 控件类型 | 说明 |
|--------|----------|------|
| 自动刷新间隔 | 下拉选择框 | 可选间隔：40s，定时自动刷新数据 |

### 4.4 时间范围

| 设置项 | 控件类型 | 说明 |
|--------|----------|------|
| 开始日期 | 日期时间选择器 | 查询的起始时间 |
| 结束日期 | 日期时间选择器 | 查询的结束时间 |

### 4.5 展示方式切换

| 方式 | 说明 |
|------|------|
| 卡片视图 | 以卡片形式展示指标 |
| 图表视图 | 以图表形式展示指标趋势 |

---

## 五、监控指标定义

### 5.1 CPU使用量

| 属性 | 说明 |
|------|------|
| 指标名称 | CPU使用量 |
| 展示形式 | 实时数值 + 趋势图 |
| 数据含义 | 任务使用的CPU资源占比 |
| 单位 | % |
| 用途 | 监控任务CPU负载 |

### 5.2 内存使用量

| 属性 | 说明 |
|------|------|
| 指标名称 | 内存使用量 |
| 展示形式 | 实时数值 + 趋势图 |
| 数据含义 | 任务使用的内存资源占比 |
| 单位 | % |
| 用途 | 监控任务内存使用，防止OOM |

---

## 六、数据展示规则

### 6.1 数据聚合
- 支持按平均值、最大值、最小值等方式聚合展示
- 默认使用平均值展示

### 6.2 多实例任务
- 分布式任务（多实例）显示聚合后的整体资源使用
- 可展开查看单个实例的资源使用情况

### 6.3 空状态展示
- 任务无监控数据时显示"暂无数据"提示
- 任务已停止时显示任务停止前的最后监控数据

---

## 七、数据刷新机制

### 7.1 自动刷新
- 按用户设置的刷新间隔自动拉取最新数据
- 默认间隔：40秒

### 7.2 切换任务刷新
- 切换任务后立即获取新任务的监控数据

---

## 八、交互流程

```
进入任务监控页面
        │
        ↓
加载默认任务（第一个运行中的任务）
        │
        ↓
获取该任务监控数据
        │
        ↓
渲染指标卡片
        │
        ├──→ 用户切换任务类型 → 过滤任务列表
        │
        ├──→ 用户切换任务 → 获取新任务数据
        │
        ├──→ 用户修改时间范围 → 获取历史数据
        │
        ├──→ 用户点击报表下载 → 导出数据文件
        │
        └──→ 自动刷新触发 → 更新数据
```
