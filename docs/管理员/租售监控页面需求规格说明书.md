# 租售监控页面需求规格说明书

## 一、页面概述

### 1.1 页面定位
租售监控页面是监控系统模块下的子页面，展示GPU资源的出租情况统计，包括总数、已出租、未出租数量及租售率，帮助管理员了解资源分配和销售情况。

### 1.2 页面功能
- 展示GPU单卡总数
- 展示已出租和未出租GPU数量
- 展示GPU租售率
- 支持时间范围查询
- 支持自动刷新
- 支持报表下载

## 三、控制区功能

### 3.1 自动刷新设置

| 设置项 | 控件类型 | 说明 |
|--------|----------|------|
| 自动刷新间隔 | 下拉选择框 | 可选间隔：40s，定时自动刷新数据 |

### 3.2 时间范围

| 设置项 | 控件类型 | 说明 |
|--------|----------|------|
| 开始日期 | 日期时间选择器 | 查询的起始时间 |
| 结束日期 | 日期时间选择器 | 查询的结束时间 |

### 3.3 展示方式切换

| 方式 | 说明 |
|------|------|
| 卡片视图 | 以卡片形式展示各项指标 |
| 图表视图 | 以图表形式展示指标趋势 |

### 3.4 报表下载

| 按钮 | 功能 |
|------|------|
| 报表下载 | 导出租售监控数据为报表文件 |

## 四、监控指标定义

### 4.1 GPU使用率

| 属性 | 说明 |
|------|------|
| 指标名称 | GPU使用率 |
| 展示形式 | 百分比数值 |
| 数据含义 | 已出租GPU占总GPU的百分比 |
| 单位 | % |
| 计算方式 | 已出租GPU数 / 总GPU数 × 100% |

|------|------|
| 指标名称 | GPU单卡 - 总数 |
| 展示形式 | 数值 |
| 数据含义 | 集群中GPU卡的总数量 |
| 单位 | 卡 |

### 4.3 GPU单卡已出租

| 属性 | 说明 |
|------|------|
| 指标名称 | GPU单卡 - 已出租 |
| 展示形式 | 数值 |
| 数据含义 | 已分配给租户的GPU卡数量 |
| 单位 | 卡 |

### 4.4 GPU单卡未出租

| 属性 | 说明 |
|------|------|
| 指标名称 | GPU单卡 - 未出租 |
| 展示形式 | 数值 |
| 数据含义 | 尚未分配的GPU卡数量 |
| 单位 | 卡 |

### 4.5 GPU租售率

| 属性 | 说明 |
|------|------|
| 指标名称 | GPU租售率 |
| 展示形式 | 百分比数值 / 进度条 |
| 数据含义 | 已出租GPU占总GPU的比率 |
| 单位 | % |
| 计算方式 | 已出租GPU数 / 总GPU数 × 100% |

## 五、业务规则

### 5.1 租售统计逻辑
- **总数**：所有已上架节点的GPU卡数之和
- **已出租**：已分配给租户的GPU卡数（白名单分配 + 用户购买）
- **未出租**：总数 - 已出租
- **租售率**：已出租 / 总数 × 100%

### 5.2 数据来源
- 数据来源于节点管理模块的节点GPU分配情况
- 实时统计，反映当前时刻的租售状态

---

## 六、数据刷新机制

### 6.1 自动刷新
- 按用户设置的刷新间隔自动拉取最新数据
- 默认间隔：40秒

### 6.2 手动刷新
- 刷新页面或重新进入页面时自动获取最新数据

---

## 七、空状态展示

当集群暂无GPU资源时，显示"暂无数据"提示，各项指标显示为0。

---

## 八、交互流程

```
进入租售监控页面
        │
        ↓
加载默认刷新间隔（40s）
        │
        ↓
获取租售统计数据
        │
        ↓
渲染指标卡片
        │
        ├──→ 用户修改时间范围 → 获取历史数据
        │
        ├──→ 用户点击报表下载 → 导出数据文件
        │
        └──→ 自动刷新触发 → 更新数据
```
