# 算网调度系统功能需求文档

## 核心概念解释

### 集群（Cluster）

**定义：** 集群是由多台物理服务器（节点）通过网络互联组成的计算资源池,作为一个统一的算力供给单元对外提供服务。集群由多个节点组成,形成"一对多"的层级结构。

### 节点（Node）

**定义：** 节点是集群中的一台物理服务器或虚拟机,是实际承载计算任务的最小硬件单元。

**业务含义说明：**

1. **"卡"的定义：** 指物理服务器中实际插载的 GPU 加速卡数量。例如"8卡"表示该物理节点是一台标准的 8-GPU 服务器（如配备 8 张 H100 或 A800 显卡）。

2. **分配类型说明：**
   - **整机分配：** 将整台物理服务器（包括所有 GPU）分配给单个租户独占使用,适用于大规模训练任务。
   - **弹性分配：** 通过容器或虚拟化技术,将节点资源动态分配给多个租户共享使用,支持灵活的资源调度。
   - **MIG分配：** 利用 NVIDIA MIG（Multi-Instance GPU）技术将单张 GPU 硬件级切分为多个独立实例,每个实例具有隔离的显存和计算资源。
   - **vGPU分配：** 通过软件虚拟化技术（如 vGPU）将 GPU 资源虚拟化后分配给多个虚拟机使用。
   - **弹性购买：** 租户按需购买弹性计算资源,按实际使用量计费,适合波动性工作负载。
   - **MIG购买：** 租户购买 MIG 切分后的 GPU 实例,获得独立的 GPU 计算单元。
   - **vGPU购买：** 租户购买虚拟化后的 GPU 资源,在虚拟机中使用 GPU 加速能力。
   - **未出售：** 节点资源尚未对外售卖,处于闲置或准备状态。

### 2. 节点运维操作逻辑

此部分定义"操作"列按钮的显示时机与业务逻辑。

#### 2.1 上架 / 下架

**上架**

- 显示时机： 当"是否上架"字段为"否"时显示"上架"按钮。
- 操作逻辑：点击"上架"按钮。系统校验节点状态是否为"就绪"。若校验通过，将节点状态更新为"已上架"，并记录当前时间为上架时间。
- 结果： 该节点变为可用状态，可被"资源分配"页面选中。

**下架**

- 显示时机： 当"是否上架"字段为"是"时显示"下架"按钮。
- 操作逻辑：点击"下架"按钮，弹出确认框："下架后该节点将无法被分配，确认下架吗？"系统校验节点上是否有正在运行的任务。若有，提示"存在运行中任务，无法下架"。若校验通过，将节点状态更新为"未上架"，并清空上架时间。
- 结果： 该节点从可用资源池中移除。

#### 2.2 维护（排空/恢复）

入口： 操作列 -> 维护。交互： 点击维护按钮后弹出"节点操作"模态框，用户需选择"排空"或"恢复"。

**排空（进入维护）**

- 显示时机："状态"为"就绪"。节点当前未处于"维护中"状态。
- 功能定义： 排空是一个主动的、保护性的隔离操作。它的核心逻辑是将目标节点从"可调度资源池"中暂时移除，并确保节点上现有的业务负载被安全地迁移或终止，从而使节点进入一个纯净的、无业务干扰的"维护中"状态。
- 详细业务逻辑：
  1. 节点标记： 系统会将该节点标记为"不可调度"，调度器不再向其分配新任务。
  2. 流量切断： 调度器会立即停止向该节点分发任何新的任务或 Pod。
  3. 负载驱逐： 系统会触发驱逐流程，优雅地终止或迁移节点上当前正在运行的所有非守护进程级的任务。
  4. 最终状态： 节点状态变更为"维护中"。

**恢复（退出维护）**

- 显示时机："状态"为"维护中"。
- 功能定义： 恢复是一个逆向操作，用于将已经完成维护的节点重新纳回到"可调度资源池"中。
- 详细业务逻辑：
  1. 节点标记： 系统会将该节点从"不可调度"改回"可调度"状态。
  2. 污点移除： 系统会自动移除之前因"排空"操作而给节点打上的"不可调度"污点。
  3. 资源重新注册： 节点会重新向调度器报告其资源可用性。
  4. 最终状态： 节点状态变更为"就绪"或"空闲"，可以接收新任务。

#### 2.3 MIG 配置 / 关闭 MIG

针对 NVIDIA A100/H100 等支持 MIG 技术的显卡，进行硬件级切分配置。

**MIG 配置**

- 显示时机（需同时满足以下条件）：
  1. 硬件支持： 节点 GPU 型号支持 MIG（如 A100, H100）。
  2. 当前模式： 分配类型为"整机分配"或"未出售"（即未开启 MIG 模式）。
  3. 节点状态： 状态为"就绪"或"未就绪"（即非"维护中"且无运行任务）。
- 操作逻辑：点击后进入 MIG 配置向导。选择切分方案： 用户从预定义列表中选择一种规格。
- 术语说明： 规格命名格式为 Xg.YGB。例如 1g.5GB。.（点号）： 表示"配置为"，即"1个计算单元组，配置为 5GB 显存"。,（逗号）： 表示"或者"，用于列举不同选项，如 1g.5GB, 2g.10GB。系统调用底层驱动接口开启 MIG 模式并创建实例。
- 结果： 分配类型变更为"MIG分配"，总GPU数量更新为 MIG 实例总数。

**关闭 MIG**

- 显示时机：分配类型为"MIG分配"。节点上无正在运行的 MIG 实例任务。
- 操作逻辑：点击后弹出确认框："关闭 MIG 将重置显卡模式，是否继续？"确认后，系统销毁所有 MIG 实例，重置 GPU 为默认模式。
- 结果： 分配类型变更为"整机分配"。

#### 2.4 资源分配

**显示时机：** "是否上架"为"是"、"状态"为"就绪"、"已分配GPU" < "总GPU"（即还有剩余资源）。

**操作逻辑：** 点击"资源分配"按钮后弹出表单弹窗，包含以下字段：

| 字段名称 | 展示形式 | 是否必填 | 字段说明与交互逻辑 |
| :-- | :-- | :-- | :-- |
| 租户 | 下拉选择框 | 是 | 过滤逻辑： 若租户未分配"租户管理员"，该选项置灰不可选，并提示："租户还没有分配租户管理员，则无法被选择"。 |
| 分配模式 | 单选卡片组 | 是 | 选项：1. 整机分配：对应物理机/裸金属。2. 弹性分配：对应容器/虚拟化实例。3. MIG实例分配：对应 GPU 硬件切分。4. vGPU实例分配：对应 GPU 软件虚拟化。交互： 选择后，下方的"资源规格"会自动过滤，只显示支持该模式的规格。 |
| 资源规格 | 下拉选择框 | 是 | 数据源： 根据"分配模式"动态加载。- 若选"整机"，显示如 8xH100-PCIE。- 若选"MIG"，显示如 1g.5GB, 2g.10GB（注意大写 GB）。 |
| CPU(核) | 数字输入框 | 否 | 默认值： 0。说明： 用户输入分配给该租户的 CPU 核数。 |
| 内存(GiB) | 数字输入框 | 否 | 默认值： 0。说明： 用户输入分配的内存大小。 |
| 包括共享存储(GB) | 数字输入框 | 否 | 默认值： 0。说明： 用户输入挂载的共享存储大小。 |
| 节点 | 下拉选择框 | 是 | 数据源： 默认选中当前发起操作的节点（如 node023）。若支持批量分配，可列出所有符合"分配模式"和"资源规格"的空闲节点。 |

**提交逻辑：** 点击"确认分配"按钮后，系统执行以下流程：

1. 预检查： 检查节点资源是否足够（再次确认未被抢占）。
2. 创建记录： 在数据库生成"资源分配记录"，关联租户ID、节点ID、规格信息。
3. 执行配置：
   - 若是整机：锁定节点，更新节点状态为"已分配"。
   - 若是弹性/MIG/vGPU：调用底层调度器（如 K8s）创建 Namespace 或实例，并应用资源配额（Quota）。
4. 反馈： 提示"分配成功"，关闭弹窗并刷新列表。

#### 2.5 资源释放

- 显示时机：节点状态异常（如"已分配"但实际无租户使用）或需要强制解绑时。
- 操作逻辑：强制清除节点上的所有绑定关系和标签，将其重置为"空闲"且"未上架"状态。
- 警告： 此操作属于高危操作，需二次密码确认。

## 第二部分：节点监控系统

### 1. 页面布局与核心组件

页面应采用清晰、模块化的仪表盘（Dashboard）布局，将不同维度的监控指标以卡片（Card）形式展示。

- **全局控制区**：位于页面顶部，包含节点选择、时间范围筛选、刷新设置。
- **监控指标区**：位于全局控制区下方，以网格形式（3列多行）排列多个监控指标卡片。

### 2. 全局控制区功能

#### 节点选择器

- 功能描述：允许用户选择需要监控的特定节点。
- 交互逻辑：支持单选或多选节点；默认显示当前选中的节点名称（如截图中的 node023）；点击下拉框展示所有可用节点的列表，建议支持搜索功能以便快速查找。

#### 时间范围筛选

- 功能描述：允许用户自定义查看数据的时间窗口。
- 交互逻辑：提供"开始日期"和"结束日期"的选择器；支持快捷选项（如：最近1小时、最近24小时、最近7天等）。

#### 自动刷新设置

- 功能描述：控制监控数据的自动更新频率。
- 交互逻辑：提供预设的时间间隔选项（如截图中的 10s、30s、40s、60s）；包含"开启"和"暂停"状态的切换开关；当处于"暂停"状态时，图表数据停止自动拉取。

### 3. 监控指标卡片功能（详细版）

每个监控指标卡片应包含指标名称、当前数值、趋势图以及单位/阈值说明。以下是各指标的详细定义与解释：

#### 3.1 GPU 核心指标

- **GPU 利用率**
  - 定义：指在过去采样周期内，GPU 计算核心（CUDA Cores/Stream Processors）处于忙碌状态的时间百分比。
  - 详细说明：反映 GPU 的计算负载情况。100% 表示计算核心完全被占用，0% 表示空闲。
  - 单位：%
- **GPU 温度**
  - 定义：GPU 板载传感器检测到的当前物理温度。
  - 详细说明：用于监控散热状况。通常包含"当前温度"和"最高限制温度"（如 85°C 或 90°C）。温度过高会触发降频保护。
  - 单位：°C (摄氏度)
- **NVLink 带宽**
  - 定义：GPU 之间通过 NVLink 高速互联总线进行数据传输的速率。
  - 详细说明：分为"发送带宽"和"接收带宽"。高带宽利用率通常出现在多卡并行训练（如大模型训练）场景中。
  - 单位：GB/s 或 MB/s
- **NVLink 健康状态**
  - 定义：NVLink 链路的连接质量与错误统计。
  - 详细说明：通常通过"重放次数"或"CRC 错误计数"来衡量。如果负数值持续增加，说明链路不稳定，可能导致多卡通信性能下降或训练中断。
  - 单位：次 (Counts) 或 状态 (正常/异常)
- **GPU 功耗**
  - 定义：GPU 当前的实时功率消耗。
  - 详细说明：反映能源消耗 and 供电压力。通常会对比"功耗上限"来展示。
  - 单位：W (瓦特)
- **GPU 显存利用率**
  - 定义：已被分配的显存占总可用显存的比例。
  - 详细说明：包含"已用显存"和"总显存"。显存溢出是导致深度学习任务失败的主要原因之一。
  - 单位：% (或 MB/GB)
- **GPU ECC 错误数**
  - 定义：ECC 显存检测到的错误计数。
  - 详细说明：分为"单比特错误"（可自动纠正）和"双比特错误"（不可纠正，严重）。该指标用于评估显存硬件的健康度，数值持续增加意味着硬件故障风险。
  - 单位：次 (Counts)
- **GPU XID 错误数**
  - 定义：NVIDIA 驱动记录的特定错误代码（XID Error Codes）。
  - 详细说明：XID 错误通常指示 GPU 陷入不可恢复的状态（如总线错误、显存错误等），往往导致应用程序崩溃或重置。监控此指标对于排查任务中断原因至关重要。
  - 单位：次 (Counts)
- **GPU Tensor Core 利用率**
  - 定义：专门用于矩阵运算的 Tensor Cores 单元的忙碌时间百分比。
  - 详细说明：针对深度学习训练和推理场景。如果该指标高而普通 CUDA 核心利用率低，说明任务主要由矩阵运算主导（如 Transformer 模型）。
  - 单位：%
- **GPU 解码/编码利用率**
  - 定义：GPU 内部专用硬件编解码单元（NVENC/NVDEC）的占用率。
  - 详细说明：反映视频处理任务的负载。在视频转码、流媒体处理或基于视频的 AI 推理中尤为重要。
  - 单位：%

#### 3.2 系统基础指标

- **内存利用率**
  - 定义：系统主内存（RAM）的已使用容量占总容量的百分比。
  - 详细说明：监控节点的系统级内存压力。内存不足会导致系统使用 Swap 分区，从而严重降低性能或导致进程被杀（OOM）。
  - 单位：%
