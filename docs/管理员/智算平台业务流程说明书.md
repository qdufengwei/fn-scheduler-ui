# 智算平台业务流程说明书

## 文档目的

本文档梳理智算平台各模块之间的业务逻辑关系，明确数据流转和操作流程。

---

## 一、平台整体架构

### 1.1 模块分层架构

```
┌─────────────────────────────────────────────────────────────────────┐
│                          用户交互层                                   │
│    监控系统  │  任务中心  │  AI资产管理  │  API KEY管理              │
├─────────────────────────────────────────────────────────────────────┤
│                          业务逻辑层                                   │
│    计费管理  │  租户资源  │  节点管理  │  vGPU/MIG配置              │
├─────────────────────────────────────────────────────────────────────┤
│                          资源调度层                                   │
│              Kubernetes  │  GPU驱动  │  存储系统                    │
├─────────────────────────────────────────────────────────────────────┤
│                          组织管理层                                   │
│               组织管理(租户/用户) → 权限控制                          │
└─────────────────────────────────────────────────────────────────────┘
```

### 1.2 核心业务实体关系

```
┌──────────────┐      1:N      ┌──────────────┐
│    租户      │──────────────→│    用户      │
└──────────────┘               └──────────────┘
       │                              │
       │ 1:N                          │ 1:N
       ↓                              ↓
┌──────────────┐               ┌──────────────┐
│   资源配额   │←──────────────│    任务      │
└──────────────┘    消耗       └──────────────┘
       │                              │
       │ N:1                          │ 使用
       ↓                              ↓
┌──────────────┐               ┌──────────────┐
│    节点      │←──────────────│  模型/数据集 │
└──────────────┘    加载        └──────────────┘
```

### 2.2 资源分配流程（白名单机制）

```
┌─────────────────────────────────────────────────────────────────────┐
│                        资源分配流程                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   触发入口：节点管理 → 资源分配 按钮                                  │
│                                                                     │
│   ┌─────────────────────────────────────────────────────────────┐   │
│   │                     资源分配表单                             │   │
│   ├─────────────────────────────────────────────────────────────┤   │
│   │  1. 选择租户                                                 │   │
│   │     └─→ 仅显示已分配租户管理员的租户                          │   │
│   │     └─→ 提示：租户还没有分配租户管理员，则无法被选择          │   │
│   │                                                              │   │
│   │  2. 选择分配模式                                              │   │
│   │     ├─→ 整机分配：整台物理服务器独占                          │   │
│   │     ├─→ 弹性分配：容器化共享使用                              │   │
│   │     ├─→ MIG实例分配：GPU硬件切分                              │   │
│   │     └─→ vGPU实例分配：GPU软件虚拟化                           │   │
│   │                                                              │   │
│   │  3. 选择资源规格                                              │   │
│   │     └─→ 根据分配模式动态过滤可选规格                          │   │
│   │     └─→ 如：8xH100-PCIE、1g.5GB等                            │   │
│   │                                                              │   │
│   │  4. 配置资源参数（可选）                                       │   │
│   │     ├─→ CPU核数                                              │   │
│   │     ├─→ 内存大小(GiB)                                         │   │
│   │     └─→ 共享存储(GB)                                          │   │
│   │                                                              │   │
│   │  5. 选择节点                                                  │   │
│   │     └─→ 显示符合规格要求的可用节点                            │   │
│   └─────────────────────────────────────────────────────────────┘   │
│                              │                                      │
│                              ↓                                      │
│   ┌─────────────────────────────────────────────────────────────┐   │
│   │                     提交分配请求                              │   │
│   │  1. 预检查：节点资源是否足够                                   │   │
│   │  2. 创建记录：在数据库生成资源分配记录                          │   │
│   │  3. 执行配置：                                                 │   │
│   │     - 整机分配：锁定节点，更新为"已分配"状态                    │   │
│   │     - 弹性/MIG/vGPU：调用K8s创建Namespace和Quota               │   │
│   │  4. 更新租户资源：增加租户的GPU配额                             │   │
│   └─────────────────────────────────────────────────────────────┘   │
│                              │                                      │
│                              ↓                                      │
│   ┌─────────────────────────────────────────────────────────────┐   │
│   │                     分配结果                                  │   │
│   │  节点管理：节点"已分配GPU"增加                                 │   │
│   │  租户资源：租户配额记录新增，显示"分配"来源                     │   │
│   │  租售监控：已出租GPU数量增加                                   │   │
│   └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 2.3 任务运行流程

```
┌─────────────────────────────────────────────────────────────────────┐
│                        任务运行流程                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   前置条件：租户已有GPU配额                                          │
│                                                                     │
│   ┌─────────────────────────────────────────────────────────────┐   │
│   │                     创建任务                                  │   │
│   │  入口：任务中心 → 创建任务                                     │   │
│   │                                                              │   │
│   │  1. 选择任务类型                                              │   │
│   │     ├─→ 训练任务：深度学习训练                                 │   │
│   │     ├─→ 模型微调：基于预训练模型微调                           │   │
│   │     ├─→ 推理服务：模型在线推理                                 │   │
│   │     └─→ 开发机：交互式开发环境                                 │   │
│   │                                                              │   │
│   │  2. 配置任务参数                                              │   │
│   │     ├─→ 选择资源规格（从租户配额中）                           │   │
│   │     ├─→ 选择镜像（从镜像仓库）                                 │   │
│   │     ├─→ 选择模型/数据集（从AI资产管理）                        │   │
│   │     ├─→ 设置实例数量                                          │   │
│   │     └─→ 设置优先级和回收策略                                  │   │
│   └─────────────────────────────────────────────────────────────┘   │
│                              │                                      │
│                              ↓                                      │
│   ┌─────────────────────────────────────────────────────────────┐   │
│   │                     资源调度                                  │   │
│   │  1. 校验：租户配额是否足够                                     │   │
│   │  2. 调度：                                                    │   │
│   │     - 有可用资源 → 调度到节点 → 状态变为"运行中"              │   │
│   │     - 无可用资源 → 进入队列 → 状态变为"排队中"                │   │
│   │  3. 记录：                                                    │   │
│   │     - 更新租户资源的"已用/总计"                               │   │
│   │     - 更新节点的"已分配GPU"                                   │   │
│   └─────────────────────────────────────────────────────────────┘   │
│                              │                                      │
│                              ↓                                      │
│   ┌─────────────────────────────────────────────────────────────┐   │
│   │                     运行监控                                  │   │
│   │  集群监控：整体GPU利用率、作业饱和度                           │   │
│   │  节点监控：任务所在节点的GPU功率、温度、显存                   │   │
│   │  任务监控：单个任务的CPU、内存使用量                           │   │
│   └─────────────────────────────────────────────────────────────┘   │
│                              │                                      │
│                              ↓                                      │
│   ┌─────────────────────────────────────────────────────────────┐   │
│   │                     任务结束                                  │   │
│   │  1. 释放资源：更新租户已用配额                                  │   │
│   │  2. 计费记录：根据运行时长和资源规格计算费用                    │   │
│   │  3. 状态更新：任务状态变为"停止"或"失败"                        │   │
│   └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## 四、关键业务规则

### 4.1 租户与资源配额

| 规则           | 说明                                   |
| -------------- | -------------------------------------- |
| 租户管理员必填 | 租户必须分配租户管理员后才能被分配资源 |
| 配额来源       | 分配（白名单机制）或购买（付费模式）   |
| 配额类型       | 整机分配、GPU卡、MIG、vGPU             |
| 配额管理       | 支持缩减资源和释放资源操作             |

### 4.2 节点状态流转

```
未就绪 ──(硬件就绪)──→ 就绪 ──(上架操作)──→ 已上架 ──(分配资源)──→ 已分配
   ↑                   │                      │                      │
   │                   │                      │                      │
   └───(维护完成)──────┴──(下架操作)──────────┴──(释放资源)──────────┘
                           │
                           ↓
                       维护中
```

### 4.3 任务状态流转

```
创建任务 → 排队中 ──(资源足够)──→ 运行中 ──→ 停止
              │                    │
              │                    ↓
              │                  失败
              │
              └──(资源不足，等待调度)
```

### 4.4 分配模式对比

| 分配模式 | 适用场景       | 资源粒度        | 隔离性           |
| -------- | -------------- | --------------- | ---------------- |
| 整机分配 | 大规模训练任务 | 整台物理服务器  | 最高（独占）     |
| 弹性分配 | 容器化应用     | 可动态调整      | 中等（容器隔离） |
| MIG分配  | 多租户共享GPU  | GPU硬件切分实例 | 高（硬件级隔离） |
| vGPU分配 | 虚拟化环境     | GPU虚拟化实例   | 中等（软件隔离） |

## 六、异常处理流程

### 6.1 节点异常处理

```
节点异常检测（状态变为"未就绪"）
        │
        ↓
┌───────────────────┐
│ 是否有运行任务？  │
└───────────────────┘
        │
   ┌────┴────┐
   ↓         ↓
  是        否
   │         │
   ↓         ↓
任务迁移    检修节点
   │         │
   ↓         ↓
恢复正常   重新上架
```

### 6.2 任务失败处理

```
任务失败
    │
    ↓
┌─────────────────────────────┐
│     失败原因分析              │
│  - 资源不足                  │
│  - 节点故障                  │
│  - 镜像/模型问题             │
│  - 代码错误                  │
└─────────────────────────────┘
    │
    ↓
根据原因处理：
  - 资源不足 → 重新排队等待
  - 节点故障 → 迁移到其他节点
  - 镜像/模型问题 → 修复后重试
  - 代码错误 → 用户自行修复
```

## 七、审计与追溯

### 7.1 审计日志记录范围

| 操作类型 | 记录内容                       |
| -------- | ------------------------------ |
| 登录     | 用户、时间、结果（成功/失败）  |
| 资源分配 | 操作者、租户、节点、规格、时间 |
| 任务操作 | 创建、删除、启动、停止         |
| 配置变更 | 服务配置、定价变更             |

### 7.2 数据追溯路径

```
问题发现
    │
    ↓
审计日志查询（按租户/用户/时间筛选）
    │
    ↓
定位操作记录
    │
    ↓
关联数据：
  - 节点状态历史
  - 任务运行记录
  - 资源使用记录
    │
    ↓
问题根因分析
```
