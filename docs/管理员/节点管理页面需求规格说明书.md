# 节点管理页面需求规格说明书

## 一、页面概述

### 1.1 页面定位

节点管理页面是集群资源模块下的核心页面，用于管理物理GPU节点的上架、下架、资源分配等操作，是资源管理的核心入口。

### 1.2 页面功能

- 展示所有物理节点的列表信息
- 支持节点上架/下架操作
- 支持资源分配操作
- 支持批量上架/下架操作
- 支持节点筛选和搜索

## 四、列表字段定义

### 4.1 字段明细

| 序号 | 字段名称 | 字段类型 | 字段说明 | 示例值 |
| --- | --- | --- | --- | --- |
| 1 | 选择框 | checkbox | 用于批量选择节点 | ✓ |
| 2 | 名称 | 文本 | 节点主机名，点击可查看详情 | cpu105 |
| 3 | 区域 | 文本 | 节点所在的物理机房或逻辑区域 | CN |
| 4 | GPU资源名称 | 文本 | GPU硬件型号 | NVIDIA-GPU-HBM2E-80GB |
| 5 | 已分配GPU | 数值 | 已分配给租户的GPU卡数 | 4 卡 |
| 6 | 总GPU | 数值 | 节点物理GPU总数 | 8 卡 |
| 7 | 分配类型 | 标签 | 资源售卖模式：整机分配、弹性分配、MIG分配、vGPU分配、未出售 | 弹性分配 |
| 8 | 状态 | 标签 | 节点状态：就绪、未就绪、维护中 | 就绪 |
| 9 | 创建时间 | 日期时间 | 节点首次接入系统的时间 | 2026-02-12 07:50:40 |
| 10 | IP地址 | 文本 | 节点内网管理IP | 10.1.2.105 |
| 11 | 是否上架 | 文本 | 是否在资源池中对外可见：是、否 | 是 |
| 12 | 上架时间 | 日期时间 | 最近一次上架操作的时间 | 2026-04-15 09:41:13 |
| 13 | 单价 | 货币 | 节点小时租赁价格 | ¥0.00 / 时 |
| 14 | 操作 | 按钮 | 上架/下架、更多操作 | 上架、下架 |

### 4.2 分配类型说明

| 分配类型 | 说明                                                |
| -------- | --------------------------------------------------- |
| 整机分配 | 整台物理服务器（所有GPU）分配给单个租户独占使用     |
| 弹性分配 | 通过容器/虚拟化技术将节点资源动态分配给多个租户共享 |
| MIG分配  | 利用NVIDIA MIG技术将GPU硬件级切分为多个独立实例     |
| vGPU分配 | 通过软件虚拟化技术将GPU资源虚拟化后分配             |
| 未出售   | 节点资源尚未对外分配，处于闲置状态                  |

### 4.3 状态说明

| 状态   | 说明                           |
| ------ | ------------------------------ |
| 就绪   | 节点正常运行，可接收任务       |
| 未就绪 | 节点异常或未完成初始化         |
| 维护中 | 节点处于维护状态，不接收新任务 |

## 五、节点操作

### 5.1 上架操作

| 属性     | 说明                                       |
| -------- | ------------------------------------------ |
| 显示时机 | "是否上架"为"否"，且状态为"就绪"           |
| 操作逻辑 | 校验节点状态 → 更新为已上架 → 记录上架时间 |
| 结果     | 节点变为可用状态，可被资源分配选中         |

### 5.2 下架操作

| 属性         | 说明                       |
| ------------ | -------------------------- |
| 显示时机     | "是否上架"为"是"           |
| 前置校验     | 检查节点上是否有运行任务   |
| 校验失败提示 | "存在运行中任务，无法下架" |
| 校验成功     | 弹出确认框 → 确认后下架    |
| 结果         | 节点从可用资源池移除       |

### 5.3 更多操作

展开更多操作菜单，包含：

- 资源分配
- 维护（排空/恢复）
- MIG配置/关闭MIG（如支持）
- 资源释放

## 六、资源分配弹窗

### 6.1 弹窗标题

资源分配

### 6.2 表单字段

| 序号 | 字段名称 | 是否必填 | 控件类型 | 说明 |
| --- | --- | --- | --- | --- |
| 1 | 租户 | 是 | 下拉选择框 | 选择要分配资源的租户 |
| 2 | 分配模式 | 是 | 单选卡片组 | 整机分配、弹性分配、MIG实例分配、vGPU实例分配 |
| 3 | 资源规格 | 是 | 下拉选择框 | 根据分配模式动态过滤可选规格 |
| 4 | CPU(核) | 否 | 数字输入框 | 分配的CPU核数，默认0 |
| 5 | 内存(GiB) | 否 | 数字输入框 | 分配的内存大小，默认0 |
| 6 | 包括共享存储(GB) | 否 | 数字输入框 | 挂载的共享存储大小，默认0 |
| 7 | 节点 | 是 | 下拉选择框 | 选择要分配的节点 |

### 6.3 租户选择限制

- 仅显示已分配租户管理员的租户
- 未分配租户管理员的租户置灰不可选
- 提示："租户还没有分配租户管理员，则无法被选择"

### 6.4 分配模式联动

- 选择分配模式后，"资源规格"下拉框自动过滤
- 整机分配：显示如 8xH100-PCIE
- MIG实例分配：显示如 1g.5GB, 2g.10GB

### 6.5 提交逻辑

1. 预检查：节点资源是否足够
2. 创建记录：数据库生成资源分配记录
3. 执行配置：
   - 整机：锁定节点，更新状态为"已分配"
   - 弹性/MIG/vGPU：调用K8s创建Namespace和Quota
4. 反馈：提示"分配成功"，关闭弹窗并刷新列表

## 八、业务规则

### 8.1 上架规则

- 状态必须为"就绪"
- 上架后节点变为可分配状态

### 8.2 下架规则

- 上架状态下可下架
- 有运行任务时禁止下架
- 下架后节点从资源池移除

### 8.3 资源分配规则

- 节点必须已上架且状态为"就绪"
- 有剩余资源（已分配GPU < 总GPU）
- 租户必须有租户管理员

## 九、交互流程

### 9.1 上架流程

```
选中未上架的节点
        │
        ↓
点击"上架"或"批量上架"
        │
        ↓
校验节点状态是否为"就绪"
        │
   ┌────┴────┐
   ↓         ↓
  否        是
   │         │
   ↓         ↓
提示异常   更新为已上架状态
          │
          ↓
       记录上架时间
          │
          ↓
       刷新列表
```

### 9.2 资源分配流程

```
点击"资源分配"按钮
        │
        ↓
弹出资源分配表单
        │
        ↓
选择租户（仅可选有管理员的租户）
        │
        ↓
选择分配模式 → 资源规格联动过滤
        │
        ↓
配置CPU、内存、存储（可选）
        │
        ↓
选择节点
        │
        ↓
点击"确认"
        │
        ↓
预检查资源是否足够
        │
   ┌────┴────┐
   ↓         ↓
  否        是
   │         │
   ↓         ↓
提示错误   创建分配记录 → 执行配置
              │
              ↓
          提示成功 → 关闭弹窗 → 刷新列表
```
