# 节点监控页面需求规格说明书

## 一、页面概述

### 1.1 页面定位

节点监控页面是监控系统模块下的子页面，提供单个物理节点的详细GPU监控指标，包括功率、温度、显存、利用率、NVLink带宽等核心指标，用于深入分析节点运行状态和排查问题。

### 1.2 页面功能

- 展示单节点GPU核心监控指标（功率、温度、利用率等）
- 展示NVLink相关指标（带宽、传输/接收字节数）
- 展示系统基础指标（CPU、内存使用量）
- 支持节点选择切换
- 支持时间范围查询
- 支持自动刷新
- 支持报表下载

## 三、控制区功能

### 3.1 节点选择器

| 设置项   | 控件类型   | 说明                                     |
| -------- | ---------- | ---------------------------------------- |
| 节点选择 | 下拉选择框 | 选择要监控的节点，默认选中第一个可用节点 |

### 3.2 自动刷新设置

| 设置项       | 控件类型   | 说明                         |
| ------------ | ---------- | ---------------------------- |
| 自动刷新间隔 | 下拉选择框 | 可选间隔：10s、30s、40s、60s |

### 3.3 时间范围

| 设置项   | 控件类型       | 说明           |
| -------- | -------------- | -------------- |
| 开始日期 | 日期时间选择器 | 查询的起始时间 |
| 结束日期 | 日期时间选择器 | 查询的结束时间 |

### 3.4 展示方式切换

| 方式     | 说明                   |
| -------- | ---------------------- |
| 卡片视图 | 以卡片形式展示各项指标 |
| 图表视图 | 以图表形式展示指标趋势 |

### 3.5 报表下载

| 按钮     | 功能                           |
| -------- | ------------------------------ |
| 报表下载 | 导出当前节点监控数据为报表文件 |

## 四、GPU核心监控指标

### 4.1 GPU功率

| 属性     | 说明                   |
| -------- | ---------------------- |
| 指标名称 | GPU功率                |
| 展示形式 | 实时数值 + 趋势图      |
| 数据含义 | GPU当前实时功率消耗    |
| 单位     | W（瓦特）              |
| 用途     | 反映能源消耗和供电压力 |

### 4.2 GPU温度

| 属性     | 说明                                 |
| -------- | ------------------------------------ |
| 指标名称 | GPU温度                              |
| 展示形式 | 实时数值 + 趋势图                    |
| 数据含义 | GPU板载传感器检测的物理温度          |
| 单位     | °C（摄氏度）                         |
| 用途     | 监控散热状况，温度过高会触发降频保护 |

### 4.3 NVLink带宽

| 属性     | 说明                              |
| -------- | --------------------------------- |
| 指标名称 | NVLink带宽                        |
| 展示形式 | 实时数值 + 趋势图                 |
| 数据含义 | GPU间NVLink互联总线的数据传输速率 |
| 单位     | GB/s 或 MB/s                      |
| 用途     | 监控多卡并行训练时的GPU间通信效率 |

### 4.4 NVLink传输字节数

| 属性     | 说明                   |
| -------- | ---------------------- |
| 指标名称 | NVLink传输字节数       |
| 展示形式 | 实时数值 + 趋势图      |
| 数据含义 | NVLink累计发送的数据量 |
| 单位     | Bytes                  |
| 用途     | 统计GPU间数据传输总量  |

### 4.5 NVLink接收字节数

| 属性     | 说明                   |
| -------- | ---------------------- |
| 指标名称 | NVLink接收字节数       |
| 展示形式 | 实时数值 + 趋势图      |
| 数据含义 | NVLink累计接收的数据量 |
| 单位     | Bytes                  |
| 用途     | 统计GPU间数据接收总量  |

### 4.6 GPU利用率

| 属性     | 说明                                |
| -------- | ----------------------------------- |
| 指标名称 | GPU利用率                           |
| 展示形式 | 实时数值 + 趋势图                   |
| 数据含义 | GPU计算核心处于忙碌状态的时间百分比 |
| 单位     | %                                   |
| 用途     | 反映GPU计算负载情况                 |

### 4.7 GPU显存利用率

| 属性     | 说明                                     |
| -------- | ---------------------------------------- |
| 指标名称 | GPU显存利用率                            |
| 展示形式 | 实时数值 + 趋势图                        |
| 数据含义 | 已分配显存占总可用显存的比例             |
| 单位     | %                                        |
| 用途     | 监控显存使用压力，显存溢出会导致任务失败 |

### 4.8 GPU SM利用率

| 属性     | 说明                      |
| -------- | ------------------------- |
| 指标名称 | GPU SM利用率              |
| 展示形式 | 实时数值 + 趋势图         |
| 数据含义 | 流多处理器(SM)的利用率    |
| 单位     | %                         |
| 用途     | 反映GPU流处理器的使用情况 |

### 4.9 GPU显存使用量

| 属性     | 说明                 |
| -------- | -------------------- |
| 指标名称 | GPU显存使用量        |
| 展示形式 | 实时数值 + 趋势图    |
| 数据含义 | 当前已使用的显存容量 |
| 单位     | GB                   |
| 用途     | 监控显存实际占用情况 |

### 4.10 GPU Tensor Core利用率

| 属性     | 说明                                                            |
| -------- | --------------------------------------------------------------- |
| 指标名称 | GPU Tensor Core利用率                                           |
| 展示形式 | 实时数值 + 趋势图                                               |
| 数据含义 | Tensor Core矩阵运算单元的利用率                                 |
| 单位     | %                                                               |
| 用途     | 针对深度学习场景，Tensor Core高利用率说明任务主要由矩阵运算主导 |

## 六、数据展示规则

### 6.1 多GPU展示

- 多卡服务器（如8卡）显示多组GPU数据
- 可选择查看单卡或全部GPU的汇总数据

### 6.2 空状态展示

- 无数据时显示"暂无数据"提示
- 节点未就绪或无GPU时，相应指标卡片显示空状态

### 6.3 图表交互

- 趋势图支持缩放和拖拽查看历史数据
- 支持鼠标悬停显示具体数值

---

## 七、数据刷新机制

### 7.1 自动刷新

- 按用户设置的刷新间隔自动拉取最新数据
- 默认间隔：40秒

### 7.2 切换节点刷新

- 切换节点选择后立即获取新节点的监控数据

---

## 八、交互流程

```
进入节点监控页面
        │
        ↓
加载默认节点（第一个可用节点）
        │
        ↓
获取该节点监控数据
        │
        ↓
渲染所有指标卡片
        │
        ├──→ 用户切换节点 → 获取新节点数据
        │
        ├──→ 用户修改时间范围 → 获取历史数据
        │
        ├──→ 用户点击报表下载 → 导出数据文件
        │
        └──→ 自动刷新触发 → 更新数据
```
