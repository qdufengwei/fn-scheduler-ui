# 集群监控页面需求规格说明书

## 一、页面概述

### 1.1 页面定位
集群监控页面是监控系统模块下的核心页面，提供整个集群的GPU资源概览，展示集群整体的资源利用率、作业饱和度、节点碎片情况等全局指标，帮助运维人员快速了解集群健康状况。

### 1.2 页面功能
- 展示集群GPU资源利用率
- 展示集群GPU卡数和未调度任务数
- 展示GPU满载节点数和碎片节点数
- 支持按资源规格筛选
- 支持时间范围查询
- 支持自动刷新
- 支持报表下载

## 三、控制区功能

### 3.1 筛选条件

| 筛选项 | 控件类型 | 说明 |
|--------|----------|------|
| 资源规格 | 下拉选择框 | 按GPU型号筛选，如NVIDIA-GPU-HBM2E-80GB |

### 3.2 自动刷新设置

| 设置项 | 控件类型 | 说明 |
|--------|----------|------|
| 自动刷新间隔 | 下拉选择框 | 可选间隔：40s，定时自动刷新数据 |

### 3.3 时间范围

| 设置项 | 控件类型 | 说明 |
|--------|----------|------|
| 开始日期 | 日期时间选择器 | 查询的起始时间 |
| 结束日期 | 日期时间选择器 | 查询的结束时间 |

### 3.4 展示方式切换

| 方式 | 说明 |
|------|------|
| 卡片视图 | 以卡片形式展示各项指标 |
| 图表视图 | 以图表形式展示指标趋势 |

### 3.5 报表下载

| 按钮 | 功能 |
|------|------|
| 报表下载 | 导出当前监控数据为报表文件 |

## 四、监控指标卡片

### 4.1 集群GPU资源利用率

| 属性 | 说明 |
|------|------|
| 指标名称 | 集群GPU资源利用率 |
| 展示形式 | 环形进度图 + 百分比数值 |
| 数据含义 | 所有节点GPU利用率的加权平均值 |
| 单位 | % |
| 计算方式 | Σ(各节点GPU利用率 × 该节点GPU数) / Σ(总GPU数) |

### 4.2 集群GPU卡数

| 属性 | 说明 |
|------|------|
| 指标名称 | 集群GPU卡数 |
| 展示形式 | 数值卡片 |
| 子指标1 | 集群GPU卡数：所有节点物理GPU卡数之和 |
| 子指标2 | 当前未调度任务数：处于Pending状态的任务数量 |
| 单位 | 卡、个 |

### 4.3 GPU节点（满载节点）

| 属性 | 说明 |
|------|------|
| 指标名称 | GPU节点 |
| 展示形式 | 数值卡片 |
| 子指标 | GPU满载节点：GPU利用率>=95%的节点数量 |
| 单位 | 个 |

### 4.4 碎片节点

| 属性 | 说明 |
|------|------|
| 指标名称 | 碎片节点 |
| 展示形式 | 数值卡片 |
| 子指标 | GPU碎片节点：有剩余资源但不足以调度新任务的节点数量 |
| 说明 | 碎片节点详情可点击查看具体节点列表 |
| 单位 | 个 |

---

## 五、数据刷新机制

### 5.1 自动刷新
- 按用户设置的刷新间隔自动拉取最新数据
- 默认间隔：40秒
- 支持暂停/恢复自动刷新

### 5.2 手动刷新
- 刷新页面或重新进入页面时自动获取最新数据

---

## 六、空状态展示

当集群暂无数据或无匹配数据时，显示"暂无数据"提示。

---

## 七、业务规则

### 7.1 GPU利用率计算
- 单节点GPU利用率：该节点所有GPU利用率的平均值
- 集群整体利用率：按GPU数量加权平均

### 7.2 满载节点判定
- GPU利用率 >= 95% 的节点判定为满载节点

### 7.3 碎片节点判定
- 节点有剩余GPU资源（已分配 < 总GPU）
- 剩余资源不足以满足最小调度单元（如1卡任务无法调度）

---

## 八、交互流程

```
进入集群监控页面
        │
        ↓
加载默认筛选条件（全部资源规格）
        │
        ↓
获取监控数据
        │
        ↓
渲染指标卡片
        │
        ├──→ 用户修改筛选条件 → 重新获取数据
        │
        ├──→ 用户修改时间范围 → 获取历史数据
        │
        ├──→ 用户点击报表下载 → 导出数据文件
        │
        └──→ 自动刷新触发 → 更新数据
```
