# 模型仓库页面需求规格说明书（租户版）

## 一、页面概述

### 1.1 页面定位

模型仓库页面展示租户可用的AI模型，包括预置模型和自定义模型，是模型选择和管理的主要入口。

### 1.2 页面功能

- 展示预置模型列表
- 展示自定义模型列表
- 查看模型详情
- 使用模型创建任务

| 操作     | 说明                 |
| -------- | -------------------- |
| 点击卡片 | 查看模型详情         |

| 创建任务 | 使用模型创建训练任务 |

---

## 六、业务规则

### 6.1 模型使用规则

- 预置模型可直接用于创建任务
- 自定义模型需先上传验证后可用
- 模型可部署为推理服务

### 6.2 模型权限规则

- 预置模型所有租户可使用
- 自定义模型仅上传租户可见

---

## 七、交互流程

### 7.1 查看模型仓库

```
进入模型仓库页面
        │
        ↓
加载预置模型标签页
        │
        ↓
显示预置模型卡片列表
        │
        ├──→ 用户切换到自定义模型 → 显示私有模型
        │
        └──→ 用户点击模型卡片 → 查看模型详情
```

### 7.2 使用模型创建任务

```
点击模型卡片
        │
        ↓
进入模型详情
        │
        ↓
点击"创建任务"或"部署服务"
        │
        ↓
跳转到任务创建页面 → 模型已预选
```

## 参考平台对齐修订（2026-05-26，原型版）

- 对齐基准：`https://aicloud.infrawaves.com/` 普通用户账号（`test01`）当前页面结构与交互。
- 原型策略：优先对齐页面信息架构、核心控件、关键按钮与状态展示；暂不接入真实后端数据。
- 交互规范：
  - 列表页统一包含筛选区、操作区、数据表格、分页。
  - 创建/编辑类操作使用右侧抽屉（Drawer）承载表单。
  - 确认类操作使用气泡确认（Popconfirm）。
- 数据口径：统计卡、趋势图、监控值、费用值当前均允许为模拟数据或空状态展示，用于产品原型演示。
- 对齐边界：若线上页面存在高耦合业务校验（鉴权、计费结算、实时监控流），原型仅保留界面入口与交互壳层。
