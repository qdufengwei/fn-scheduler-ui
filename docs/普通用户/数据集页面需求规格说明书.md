# 数据集页面需求规格说明书（租户版）

## 一、页面概述

### 1.1 页面定位

数据集页面用于管理租户的训练和推理数据集，支持上传、版本管理和删除，是AI工作流的数据管理入口。

### 1.2 页面功能

- 展示数据集列表
- 创建新数据集
- 添加数据集版本
- 删除数据集

---

## 八、业务规则

### 8.1 数据集创建规则

- 数据集名称在同一租户下唯一
- 支持多种数据格式上传

### 8.2 数据集使用规则

- 数据集可挂载到训练任务
- 数据集可挂载到推理服务
- 数据格式需符合任务要求

### 8.3 版本管理规则

- 支持多版本管理
- 可选择使用指定版本

## 九、交互流程

### 9.1 创建数据集

```
点击"新建数据集"
        │
        ↓
弹出创建表单
        │
        ↓
填写数据集名称
        │
        ↓
上传数据文件
        │
        ↓
点击"确认"
        │
        ↓
开始上传 → 显示进度 → 完成后刷新列表
```

### 9.2 添加版本流程

```
点击"添加版本"
        │
        ↓
弹出添加版本表单
        │
        ↓
上传新版本数据文件
        │
        ↓
点击"确认" → 上传完成 → 更新版本号
```

## 参考平台对齐修订（2026-05-26，原型版）

- 对齐基准：`https://aicloud.infrawaves.com/` 普通用户账号（`test01`）当前页面结构与交互。
- 原型策略：优先对齐页面信息架构、核心控件、关键按钮与状态展示；暂不接入真实后端数据。
- 交互规范：
  - 列表页统一包含筛选区、操作区、数据表格、分页。
  - 创建/编辑类操作使用右侧抽屉（Drawer）承载表单。
  - 确认类操作使用气泡确认（Popconfirm）。
- 数据口径：统计卡、趋势图、监控值、费用值当前均允许为模拟数据或空状态展示，用于产品原型演示。
- 对齐边界：若线上页面存在高耦合业务校验（鉴权、计费结算、实时监控流），原型仅保留界面入口与交互壳层。
