# 任务监控页面需求规格说明书（租户版）

## 一、页面概述

### 1.1 页面定位

任务监控页面用于监控租户任务的运行状态和资源使用情况，提供详细的性能指标和报表。

### 1.2 页面功能

- 展示任务运行状态
- 监控GPU/CPU/内存使用情况
- 查看GPU温度、功率、利用率等指标
- 下载监控报表

## 七、业务规则

### 7.1 监控数据规则

- 仅监控运行中的任务
- 数据按设定间隔自动刷新
- 支持历史数据查询

### 7.2 数据展示规则

- 排队中的任务暂无监控数据
- 任务结束后保留历史数据

---

## 八、交互流程

### 8.1 查看任务监控

```
进入任务监控页面
        │
        ↓
选择要监控的任务
        │
        ↓
设置筛选条件和时间范围
        │
        ↓
显示监控指标图表
        │
        ├──→ 调整刷新间隔 → 自动更新数据
        │
        └──→ 点击"报表下载" → 下载监控数据
```

### 8.2 切换监控任务

```
点击任务类型下拉框
        │
        ↓
选择任务类型
        │
        ↓
选择具体任务名称
        │
        ↓
加载该任务的监控数据
```

## 参考平台对齐修订（2026-05-26，原型版）

- 对齐基准：`https://aicloud.infrawaves.com/` 普通用户账号（`test01`）当前页面结构与交互。
- 原型策略：优先对齐页面信息架构、核心控件、关键按钮与状态展示；暂不接入真实后端数据。
- 交互规范：
  - 列表页统一包含筛选区、操作区、数据表格、分页。
  - 创建/编辑类操作使用右侧抽屉（Drawer）承载表单。
  - 确认类操作使用气泡确认（Popconfirm）。
- 数据口径：统计卡、趋势图、监控值、费用值当前均允许为模拟数据或空状态展示，用于产品原型演示。
- 对齐边界：若线上页面存在高耦合业务校验（鉴权、计费结算、实时监控流），原型仅保留界面入口与交互壳层。
