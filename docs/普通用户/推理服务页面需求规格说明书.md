# 推理服务页面需求规格说明书（租户版）

## 一、页面概述

### 1.1 页面定位

推理服务页面用于管理模型的在线推理服务部署，提供实时推理API能力，是模型对外提供服务的主要入口。

### 1.2 页面功能

- 展示推理服务列表
- 创建推理服务
- 批量删除服务
- 管理服务副本数
- 查看服务状态

## 四、推理服务列表字段

### 4.1 字段明细

| 序号 | 字段名称 | 字段类型 | 字段说明 | 示例值 |
| --- | --- | --- | --- | --- |
| 1 | 选择框 | checkbox | 用于批量选择服务 | ✓ |
| 2 | 名称/ID | 文本 | 服务名称和唯一ID | my-inference / svc-xxx |
| 3 | 模型名称 | 文本 | 部署的模型名称 | Qwen3-4B |
| 4 | 推理框架 | 标签 | 推理框架类型 | vLLM、TGI |
| 5 | 状态 | 标签 | 运行中、停止、排队中、失败 | 运行中 |
| 6 | 就绪状态 | 文本 | 就绪实例数/总实例数 | 1/1 |
| 7 | 创建时间 | 日期时间 | 服务创建时间 | 2026-05-23 15:29:21 |
| 8 | GPU数量 | 数值 | 服务使用的GPU卡数 | 1 |
| 9 | 资源规格 | 文本 | GPU规格名称 | NVIDIA-GPU-HBM2E-80GB |
| 10 | 当前副本数 | 数值 | 当前运行的副本数 | 1 |
| 11 | 实例数量 | 数值 | 配置的实例总数 | 1 |
| 12 | 用户 | 文本 | 服务创建者 | test01 |
| 13 | 操作 | 按钮组 | 查看、停止、删除等 | [查看] [停止] |

### 4.2 推理框架说明

| 框架         | 说明                      |
| ------------ | ------------------------- |
| vLLM         | 高性能大模型推理框架      |
| TGI          | Text Generation Inference |
| TensorRT-LLM | NVIDIA推理优化框架        |

### 4.3 服务状态说明

| 状态   | 说明                         |
| ------ | ---------------------------- |
| 运行中 | 推理服务正常运行，可接收请求 |
| 停止   | 服务已停止                   |
| 排队中 | 等待资源部署                 |
| 失败   | 服务部署失败                 |

## 七、业务规则

### 7.1 推理服务规则

- 服务名称在同一租户下唯一
- 必须选择模型和推理框架
- 副本数可动态调整

### 7.2 资源分配规则

- GPU规格需匹配模型需求
- 资源不足时服务进入排队状态

### 7.3 API访问规则

- 推理服务通过API KEY鉴权
- 支持配置访问权限

## 八、交互流程

```
点击"创建推理服务"
        │
        ↓
进入创建页面
        │
        ↓

        │
        ↓
配置资源和副本数
        │
        ↓
配置API访问
        │
        ↓
点击"确认" → 创建服务 → 刷新列表
```

### 8.2 调整副本数

```
点击服务详情
        │
        ↓
进入服务详情页
        │
        ↓
调整副本数配置
        │
        ↓
点击"确认" → 更新服务 → 刷新状态
```

## 参考平台对齐修订（2026-05-26，原型版）

- 对齐基准：`https://aicloud.infrawaves.com/` 普通用户账号（`test01`）当前页面结构与交互。
- 原型策略：优先对齐页面信息架构、核心控件、关键按钮与状态展示；暂不接入真实后端数据。
- 交互规范：
  - 列表页统一包含筛选区、操作区、数据表格、分页。
  - 创建/编辑类操作使用右侧抽屉（Drawer）承载表单。
  - 确认类操作使用气泡确认（Popconfirm）。
- 数据口径：统计卡、趋势图、监控值、费用值当前均允许为模拟数据或空状态展示，用于产品原型演示。
- 对齐边界：若线上页面存在高耦合业务校验（鉴权、计费结算、实时监控流），原型仅保留界面入口与交互壳层。
