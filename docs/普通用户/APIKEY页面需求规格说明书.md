# API KEY页面需求规格说明书（租户版）

## 一、页面概述

### 1.1 页面定位

API KEY页面用于管理租户的API访问密钥，用于推理服务的外部调用鉴权，是API访问的安全入口。

### 1.2 页面功能

- 展示API KEY统计信息
- 展示API KEY列表
- 创建新的API KEY
- 编辑API KEY权限
- 禁用/启用API KEY

## 九、业务规则

### 9.1 API KEY生成规则

- KEY ID自动生成，不可修改
- KEY格式为随机字符串
- 创建后立即可用

### 9.2 权限控制规则

- 不同权限对应不同的API访问范围
- 读取权限：可查询推理服务状态和结果
- 写入/执行权限：可提交推理请求

### 9.3 禁用规则

- 禁用后KEY立即失效
- 禁用的KEY可重新启用

## 参考平台对齐修订（2026-05-26，原型版）

- 对齐基准：`https://aicloud.infrawaves.com/` 普通用户账号（`test01`）当前页面结构与交互。
- 原型策略：优先对齐页面信息架构、核心控件、关键按钮与状态展示；暂不接入真实后端数据。
- 交互规范：

  - 确认类操作使用气泡确认（Popconfirm）。
- 数据口径：统计卡、趋势图、监控值、费用值当前均允许为模拟数据或空状态展示，用于产品原型演示。
