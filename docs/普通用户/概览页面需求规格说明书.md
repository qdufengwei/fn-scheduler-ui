# 概览页面需求规格说明书（租户版）

## 一、页面概述

### 1.1 页面定位
概览页面是租户用户登录后的首页，展示租户的核心业务数据和资源使用情况，是租户了解自身资源状态和任务运行情况的入口。

### 1.2 页面功能
- 展示任务数量统计
- 展示计费信息
- 展示消息通知
- 展示资源概览

## 七、业务规则

### 7.1 数据展示规则
- 仅展示当前租户的数据
- 数据实时刷新（或定时刷新）
- 任务数量统计包含所有状态的任务

### 7.2 计费展示规则
- 费用按租户维度统计
- 支持切换不同计费周期查看
- 费用精确到小数点后两位

### 7.3 权限规则
- 租户用户只能查看本租户数据
- 普通用户只能查看自己的资源使用情况

## 八、交互流程

### 8.1 查看概览

```
租户用户登录系统
        │
        ↓
加载概览页面
        │
        ↓
显示任务数量、计费、消息、资源概览
        │
        ├──→ 用户点击任务卡片 → 跳转到对应任务列表页
        │
        ├──→ 用户切换计费周期 → 更新费用数据
        │
        └──→ 用户点击消息 → 查看消息详情
```

### 8.2 跳转任务详情

```
点击任务类型卡片
        │
        ↓
跳转到任务中心对应标签页
        │
        ↓
显示该类型的任务列表
```

## 参考平台对齐修订（2026-05-26，原型版）

- 对齐基准：`https://aicloud.infrawaves.com/` 普通用户账号（`test01`）当前页面结构与交互。
- 原型策略：优先对齐页面信息架构、核心控件、关键按钮与状态展示；暂不接入真实后端数据。
- 交互规范：
  - 列表页统一包含筛选区、操作区、数据表格、分页。
  - 创建/编辑类操作使用右侧抽屉（Drawer）承载表单。
  - 确认类操作使用气泡确认（Popconfirm）。
- 数据口径：统计卡、趋势图、监控值、费用值当前均允许为模拟数据或空状态展示，用于产品原型演示。
- 对齐边界：若线上页面存在高耦合业务校验（鉴权、计费结算、实时监控流），原型仅保留界面入口与交互壳层。
