算网调度系统

智算平台业务流程详细梳理

一、平台概述

算网调度系统是一个面向GPU算力资源的智能调度与管理平台，为AI训练、模型推理、模型微调、交互式开发等场景提供GPU算力资源的分配、调度和监控能力。平台覆盖从物理GPU节点管理到AI任务执行的全链路算力资源生命周期管理，采用"管理员-租户"两级角色体系，通过白名单机制实现资源的精准分配。

1.1 平台定位与核心能力

多模式资源分配
支持整机分配、弹性分配、MIG分配、vGPU分配四种模式。整机分配适合大规模训练独占资源，弹性分配支持容器化共享，MIG/vGPU实现GPU资源的精细化切分，满足不同场景的资源隔离需求。
全链路任务管理
一站式管理训练任务、模型微调、推理服务、开发机四种任务类型。支持任务创建、调度、监控、停止全流程，内置任务模板和回收站机制，保障任务数据安全。
多维度实时监控
覆盖集群、节点、任务三级监控体系。监控GPU利用率、温度、显存、功耗、NVLink带宽等核心指标，支持自动刷新和报表下载，助力运维决策。
精细化计费体系
支持按需计费、包月、包年多种计费模式。按需计费精确到秒级，包年包月享受折扣优惠，灵活的定价策略满足不同租户需求。
AI资产生态
内置预置模型、数据集、镜像仓库。预置Qwen、Llama、DeepSeek等主流模型，支持自定义模型和数据集上传，为AI任务提供完整资源支撑。
API服务能力
推理服务部署后通过API KEY对外提供API能力。支持API KEY的创建、管理和权限控制，方便与外部系统集成。

二、角色与权限体系

2.1 角色定义

角色 核心职责 职责总结
系统管理员
  • 管理物理节点：上架/下架、维护、MIG配置
  • 管理租户：创建租户、分配资源、设置折扣
  • 运维监控：集群监控、节点监控、租售监控
  • 计费管理：定价配置、收益查看
  • 系统配置：服务配置、审计日志查看
 平台运营者：负责算力资源的供给、分配、监控和计费，确保平台稳定运行和收益统计
普通用户
  • 创建和管理任务：训练、微调、推理、开发机
  • 管理AI资产：模型、数据集、镜像
  • 查看资源使用情况
  • 管理API KEY
  • 查看账单和费用
 资源使用者：利用分配的算力资源执行AI任务，管理任务生命周期和AI资产

2.2 权限边界说明

关键权限规则：
  • 租户必须分配租户管理员后才能被分配GPU资源
  • 管理员可查看所有租户的任务和资源，普通用户只能查看本租户数据
  • 资源分配采用白名单机制，由管理员主动分配，非租户自主申请
  • 计费账单对管理员可见全部，对普通用户仅可见本租户账单

2.3 管理员功能菜单

监控模块
核心集群监控
查看集群整体GPU资源利用率、GPU卡数、未调度任务数、满载节点数、碎片节点等关键指标。帮助管理员掌握集群整体负载状况，识别资源瓶颈和调度问题。

查看单节点的GPU功率、温度、显存、利用率、NVLink带宽等详细指标，支持卡片/图表视图切换。帮助管理员深入分析节点运行状态，定位硬件异常和性能问题。
核心租售监控
统计GPU单卡总数、已出租数、未出租数、租售率。帮助管理员了解资源出租情况，优化资源配置和定价策略。
辅助任务监控
跨租户查看所有任务的运行状态和资源使用情况。帮助管理员监控任务执行情况，处理异常任务。

核心节点管理
管理物理GPU节点的上架/下架、维护、MIG配置、资源分配、资源释放。帮助管理员完成节点生命周期管理，将物理资源纳入平台调度。

辅助vGPU管理

查看登录、任务、资源、配置等操作行为记录。帮助管理员追溯操作历史，排查问题，满足安全审计需求。
辅助服务配置
设置内存/CPU上限、默认镜像、日志配置等全局参数。帮助管理员统一配置平台运行参数，确保系统稳定运行。
计费管理模块
核心商品定价
配置GPU和存储资源的基础价格、包月折扣、包年折扣。帮助管理员制定灵活的定价策略，实现收益最大化。
核心收益
查看账单概览（日/周/月/季度/年）、总金额、使用时长、已支付金额。帮助管理员了解平台收益情况，进行经营分析。
任务中心模块
辅助最新任务

辅助任务模板
查看管理保存的任务模板。帮助管理员了解常用任务配置，优化资源配置建议。
辅助任务回收站
查看已删除的任务记录。帮助管理员追溯历史任务，处理数据恢复需求。
AI资产管理模块
辅助模型仓库
管理预置模型（Qwen/Llama/DeepSeek）和自定义模型。帮助管理员为租户提供开箱即用的模型资源，降低使用门槛。
辅助模型服务市场
管理模型即服务（MaaS）配置。帮助管理员提供模型API服务，拓展平台服务能力。
辅助数据集
管理训练/推理数据集。帮助管理员为租户提供数据集资源，支持AI任务执行。
辅助镜像仓库
管理预置镜像（Jupyter/VSCode/BusyBox）和自定义镜像。帮助管理员提供运行环境镜像，确保任务正常运行。
组织管理模块
核心租户管理
创建、编辑、删除租户，设置租户折扣。帮助管理员管理租户账户，建立租户体系。
核心用户管理
创建、编辑、禁用用户，分配用户角色。帮助管理员管理平台用户，控制用户权限。

  • 建议先完成定价配置再进行资源分配

3.5 监控管理流程

监控管理提供集群、节点、任务三级监控体系，帮助管理员实时掌握平台运行状态，及时发现和处理异常。支持自动刷新、报表下载、历史数据查询等功能。

监控指标体系

监控层级 监控页面 核心指标 解决问题
集群级 集群监控 GPU利用率、作业饱和度、满载节点、碎片节点 帮助管理员掌握整体负载，优化资源配置

租户与资源配额规则

规则 说明
租户管理员必填 租户必须分配租户管理员后才能被分配GPU资源
配额来源 分配（白名单机制，管理员主动分配）或购买（付费模式）
配额类型 整机分配、GPU卡、MIG、vGPU
配额管理 支持缩减资源和释放资源操作

节点管理规则

规则 说明
上架条件 节点状态必须为"就绪"才能上架
下架条件 节点上无运行任务时才能下架
维护模式 维护中节点不接收新任务，现有任务会被驱逐
MIG配置 仅支持NVIDIA A100/H100等支持MIG的GPU

任务调度规则

规则 说明
优先级调度 高优先级任务优先调度（1-6，数值越大优先级越高）
资源不足 任务进入排队状态，按创建顺序排队
资源释放 任务结束后释放资源，更新租户已用配额

计费规则

规则 说明
按需计费 费用 = 运行时长 × GPU单价 × GPU数量
包年/包月 预付费固定周期计费，享受折扣
定价生效 修改后立即生效，新订单应用新价格

异常处理规则

节点异常处理：
  • 节点状态变为"未就绪"时，检查是否有运行任务
  • 有任务 → 任务迁移到其他节点
  • 无任务 → 检修节点 → 重新上架
任务失败处理：
  • 资源不足 → 重新排队等待
  • 节点故障 → 迁移到其他节点
  • 镜像/模型问题 → 修复后重试
  • 代码错误 → 用户自行修复
