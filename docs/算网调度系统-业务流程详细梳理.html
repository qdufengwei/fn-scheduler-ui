一、平台概述

算网调度系统是一个面向GPU算力资源的智能调度与管理平台，为AI训练、模型推理、模型微调等场景提供GPU算力资源的分配、调度和监控能力。

核心定位：覆盖从物理GPU节点管理到AI任务执行的全链路算力资源生命周期管理

1.1 平台特色

多模式资源分配
支持整机分配、弹性分配、MIG分配、vGPU分配四种模式
全链路任务管理
训练任务、模型微调、推理服务、开发机一站式管理
实时监控告警
GPU利用率、温度、显存、功耗等多维度监控
精细化计费
按需计费、包年包月多种计费模式

6.2 节点运维操作

操作 触发条件 业务逻辑
上架 状态=就绪，未上架 校验就绪 → 更新为已上架 → 记录上架时间
下架 已上架 检查无运行任务 → 确认下架 → 从资源池移除
维护（排空） 状态=就绪 标记不可调度 → 驱逐负载 → 状态变为"维护中"
恢复 状态=维护中 移除污点 → 重新注册资源 → 状态变为"就绪"
MIG配置 支持MIG的GPU，整机/未出售 选择切分方案 → 调用驱动 → 变为MIG分配
关闭MIG 当前为MIG分配 销毁实例 → 重置GPU → 变为整机分配
资源分配 已上架+就绪+有剩余资源 选择租户/模式/规格 → 预检 → 创建分配记录
资源释放 状态异常需强制解绑 二次密码确认 → 清除绑定 → 重置为空闲

7.1 分配模式对比

分配模式 适用场景 资源粒度 隔离性
整机分配 大规模训练任务 整台物理服务器 最高（独占）
弹性分配 容器化应用 可动态调整 中等（容器隔离）
MIG分配 多租户共享GPU GPU硬件切分实例 高（硬件级隔离）
vGPU分配 虚拟化环境 GPU虚拟化实例 中等（软件隔离）

十三、关键业务规则

13.1 租户与资源配额

规则 说明
租户管理员必填 租户必须分配租户管理员后才能被分配资源
配额来源 分配（白名单机制）或购买（付费模式）
配额类型 整机分配、GPU卡、MIG、vGPU
配额管理 支持缩减资源和释放资源操作

13.2 计费规则

规则 说明
按需计费 费用 = 运行时长 × GPU单价 × GPU数量
包年/包月 预付费固定周期计费，享受折扣
定价生效 修改后立即生效，新订单应用新价格

13.3 任务调度规则

规则 说明
优先级调度 高优先级任务优先调度（1-6，数值越大优先级越高）
资源不足 任务进入排队状态，按创建顺序排队
资源释放 任务结束后释放资源，更新租户已用配额

13.4 异常处理

节点异常处理：
  • 节点状态变为"未就绪"时，检查是否有运行任务
  • 有任务 → 任务迁移到其他节点
  • 无任务 → 检修节点 → 重新上架
任务失败处理：
  • 资源不足 → 重新排队等待
  • 节点故障 → 迁移到其他节点
  • 镜像/模型问题 → 修复后重试
  • 代码错误 → 用户自行修复

13.5 审计日志记录范围

操作类型 记录内容
登录用户、时间、结果（成功/失败）
资源分配操作者、租户、节点、规格、时间
任务操作创建、删除、启动、停止
配置变更服务配置、定价变更

十四、总结

该算网调度系统是一个完整的GPU算力资源管理平台，核心业务流程如下：

1. 管理员初始化
创建租户和用户 → 接入物理节点 → 配置定价和服务 → 节点上架
2. 资源分配
通过白名单机制将GPU资源分配给租户（整机/弹性/MIG/vGPU四种模式）
3. 任务执行
租户使用分配的资源创建训练/微调/推理/开发任务
4. 监控计费
实时监控资源使用情况，按使用时长计费生成账单
5. AI资产管理
管理模型、数据集、镜像，为任务提供资源支撑
6. API服务
推理服务部署后通过API KEY对外提供API能力
项目定位：前端原型项目，不调用后台接口，所有页面数据使用Mock数据。基于 Vue 3 + TypeScript + Vite + Pinia + Ant Design Vue 构建，采用 Vben Admin 框架。
